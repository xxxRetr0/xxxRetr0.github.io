MMZFM422K5HDASKDN5TVU3SKOZRFGQRRMMZFM6KJJBSG6WSYJJWESSCWPJNFQSTVLFLTC3CJIQYGOSTZKJ2VSVZRNRFHOPJ5，

c2VsZWN0ICogZnJvbSB1c2VyIHdoZXJlIHVzZXJuYW1lID0gJyRuYW1lJw==

2、easy_tornado
打开来发现三个文件夹，点开来是这样子👇👇👇

第一个是应该就是存放 flag 的文件夹，但是不太一样（fllllllllllllag）hhhhh
第二个不及丢是什么，一查得知是个渲染函数
第三个是一个嵌套的 md5 格式，应该就是通往最终的flag了

这个公式里的 filename 我们是知到了的，差个 cookie_secret

先试看看能不能直接进去？（估计不能）
/flie?filename=/fllllllllllllag

跳转到一个报错页面，且发现 url 里的 Error 显示在页面中，尝试改变它，发现欸还真可以

或许正是操作这个变量以获得 cookie_secret

那还有什么办法？这应该就需要第二个文件里的 render 了

搜哇！！！

render 确实和 cookie_secret 有些关系，（大佬们）查到 https://www.tornadoweb.org/en/latest/index.html

我看不懂55555，借来大佬构建的函数得到 cookie_secret
函数：/error?msg= （不过确实是从这点突破哈哈哈哈）

于是拿到 cookie_secret 👇👇👇

接下来就简单了……吧
将 filename 也就是 fllllllllllllag 加密 MD5 再把结果与 f0ac78e3-e3cb-4ba5-95b4-c6d2f1cf9f85 组合一起加密 MD5 然后通过 /file?filename=/fllllllllllllag&filehash=即可得到 flag
我以为这样子就可以结束了，可是无论我怎么搞怎么搞得到的 MD5 码都不行，试了大半个小时愣是解不出来
无奈去找大佬 wp 拿到脚本跑出结果，就纳闷为啥他的就行，然后发现！！！👇👇👇


import hashlib

hash = hashlib.md5() #生成一个md5对象
filename = ‘/fllllllllllllag’
cookie = ‘f0ac78e3-e3cb-4ba5-95b4-c6d2f1cf9f85’

hash.update(filename.encode(‘utf-8’)) #update中的参数必须是二进制的字节流
s = hash.hexdigest() #hexdigest是用来返回32位的十六进制字符串
print(s)

hash = hashlib.md5()
hash.update((cookie+s).encode(‘utf-8’))
print(hash.hexdigest())

原来 / 也是文件名的一部分！！！！完整文件名应该是 /fllllllllllllag ，一直以为只有当 url 中用到文件名才会用到那个 / 来着
最后得到 flag ……
