OSI七层协议和TCP/IP五层协议对应协议关系

1、基于OSI模型扫描优缺点
二层扫描 优：速度快，可靠
缺：不可路由
三层扫描 优：可路由，速度快
缺：相对于二层慢，易被边界防火墙过滤
四层扫描 优：可路由，可靠，不易被防火墙过滤，可发现所有端口被过滤的主机
缺：可能被基于状态过滤的防火墙过滤，全端口扫描速度慢

3、ARPING
ARP协议将局域网 IP 地址转为网卡的 MAC 地址（物理地址）
可用于检查局域网中 IP 是否有冲突

┌──(root㉿kali)-[~]
└─# arping 192.168.1.1 -c 1
ARPING 192.168.1.1
60 bytes from cc:08:fb:78:b3:35 (192.168.1.1): index=0 time=2.600 msec

— 192.168.1.1 statistics —
1 packets transmitted, 1 packets received, 0% unanswered (0 extra)
rtt min/avg/max/std-dev = 2.600/2.600/2.600/0.000 ms

ARPING批量检查脚本👇👇👇
#!/bin/bash
if [ “$#” -ne 1 ];then #-ne 1 参数不等于为1
echo “Usage - ./arping.sh [interface]”
echo “Excample - ./arping.sh eth0”
echo “Example will perform an ARP scan of the local subnet to which eth0 is assigned”
exit
fi

interface=$1 #输入的一个值，，赋值给interface变量
prefix=$(ifconfig $interface | grep “broadcast” | cut -d “ “ -f 10 | cut -d ‘.’ -f 1-3) #取IP地址的前缀，如：192.168.1
#echo “$prefix”
for addr in $(seq 98 110);do
arping -c 1 $prefix.$addr -I $interface | grep “reply from” | cut -d” “ -f 4
done

┌──(root㉿kali)-[~]
└─# chmod +x arping1.sh #对 arping1.sh 提权

┌──(root㉿kali)-[~]
└─# ./arping1.sh eth0 #对与IP地址相同开头的地址进行检测

^C^Z #退出当前操作（检测会从 .0 开始到 .255 ，真的好久，溜了溜了）

4、Netdiscover

选项说明
-i device 网络接口
-r range 扫描范围，例如192.168.0.0/24，仅支持/8, /16和/24
-l file 扫描范围列表文件，每行一个范围
-p 被动模式：不发生任何报文，仅嗅探
-m file 扫描已知Mac地址和主机名的列表文件
-F filter 自定义pcap filter表达式(默认“arp”)
-s time 每个ARP请求间的休眠时间(毫秒,默认1ms)
-c count 发送每个ARP请求的次数，用于丢失数据包的网络，默认1次
-n node 扫描最后的源IP，默认为67(x.x.x.67),允许范围为2-253
-d 忽略自动扫描和快速模式的主配置文件,扫描默认的范围和IP
-f 启用快速模式扫描,扫描每个网段的.1、.100 和 .254
-P 产生输出到文件或者其他解析程序，扫描完成后退出，例如：netdiscover -P -r 192.168.20.0/16 | grep 192.168.20.100
-L 与-P类似，但在主动扫描后继续捕获ARP包
-N 不打印表头，仅在-P 或者-L启用时有效

主动模式
主动探测发现网络内主机，但容易被发现
┌──(root㉿kali)-[~]
└─# netdiscover -i eth0 -r 192.168.1.0/24

被动模式
更加隐蔽，但速度较慢，网卡被设置为混杂模式来侦听网络内的 arp 数据包进行被动侦测（只有网络内设备发送的 arp 包才能被探测）
┌──(root㉿kali)-[~]
└─# netdiscover -p

5、hping3
每次只扫描一个目标，也可进行 DOS 攻击的实验
对 http://www.xuegod.cn/ 进行压力测试
┌──(root㉿kali)-[~]
└─#hping3 -c 1000 -d 120 -S -w 64 -p 80 –flood –rand-source xuegod.cn

6、fping
可理解为 ping 命令的加强版，进行网段的扫描
┌──(root㉿kali)-[~]
└─#fping -g 192.168.1.0/24 -c 1 >fping.txt

fping.txt 指将扫描结果重定向到 fping.txt 文件中，如果该网段输出结果很多，则文件中只会保留存活的主机信息
