八、distinct 去重

distinct 只能放在前面，否则会出错，当 distinct 后有多个字段时，是指后面多个字段联合起来去重

统计岗位的数量？
select count(distinct job) from emp;

九、连接查询

1、分类
按照语法年代分：①SQL92
②SQL99
按照表的连接方式分：①内连接：
等值连接
非等值连接
自连接

                                 ②外连接：
                                               左外连接（左连接）
                                               右外连接（右连接）
                                               全连接

2、在两表连接时若不做任何处理，会出现笛卡尔积现象👇👇👇

EMP表①
DEPT表②
联合表③

……

即当两张表进行连接查询的时候，没有任何条件进行限制，最终的查询结果条数是两张表记录条数的乘积。

对条件进行限制可以避免现象， select ename,dname from emp,dept where emp.deptno = dept.deptno;
这里计算机仍执行了56次，经过筛选得出14个结果，得知连接的表越多，执行效率越低

3、内连接–等值连接

select … from a inner join b on 连接条件 where 筛选条件 ；//inner可省略，条件为等量关系，因此叫等值连接

mysql> select e.ename,d.dname from emp e,dept d where e.deptno = d.deptno;//SQL92
缺点：表的连接、后面的筛选都放到了where后面

mysql> select e.ename,d.dname from emp e join dept d on e.deptno = d.deptno;//SQL99
优点：表的连接独立，且on为筛选，若需进一步筛选可加where

4、内连接–非等值连接

select … from a inner join b on c between d and e;//inner可省略，条件为非等量关系，因此叫非等值连接

mysql> select e.ename,e.sal,s.grade from emp e join salgrade s on e.sal between s.losal and s.hisal;

5、内连接–自连接

将一张表视为两张相同的表，分别命名

//查每个员工各自的上级领导👇👇👇
mysql> select e1.ename as ‘员工’,e2.ename as ‘领导’ from emp e1 join emp e2 on e1.mgr = e2.empno;

6、外连接

select … from a left/right outer join b on …;//左/右外连接， outer 可省略,外连接最重要的特点是：主表的数据无条件的全部查询出来。

将 join 左/右边的表当作主表，将主表的内容全查出来，顺带着把符合条件的另一个表中的内容查出来，若主表没有对应的内容，这这个对应的部分显示为 NULL

如：select e.ename,d.dname from dept d right join emp e on e.deptno = d.deptno;//👇👇👇

7、三、四张表连接

select … from a join b on a和b连接条件 join c on a和c连接条件 join d on a和d连接条件；//一条sql语句中内外连接可一起用

如：查员工名字、对应领导名字、部门名、工资和工资等级👇👇👇
mysql> select e.ename,e1.ename as ldename,e.sal,d.dname,s.grade from emp e join dept d on e.deptno = d.deptno join salgrade s on e.sal between s.losal and s.hisal left join emp e1 on e.mgr = e1.empno;

十、子查询

1、select语句当中嵌套select语句，被嵌套的select语句是子查询

2、子查询出现的位置
select …(select)… from …(select)… where …(select)…

3、where 语句中的子查询
如：查工资不是最低的所有人名
错误：mysql> select * from emp where sal > avg(sal); //错误的写法，where后面不能直接使用分组函数。
正确：mysql> select ename from emp where sal > (select min(sal) from emp);//👇①

  from 语句中的子查询
  如：查各岗位平均工资的工资等级
  错误：mysql> select j.job,s.grade from (select job,avg(sal) from emp group by job) j join salgrade s on j.avg(sal) between s.losal and s.hisal;//未给 avg(sal) 起别名，导致后面的 avg(sal) 被判定为一个分组函数，语句错误
  正确：mysql> select j.job,s.grade from (select job,avg(sal) as avgsal from emp group by job) j join salgrade s on j.avgsal between s.losal and s.hisal;//👇👇②

  select 语句中的子查询
  如：查员工对应的部门名字
  ①mysql> select e.ename,d.dname from emp e,dept d where e.deptno = d.deptno;
  ②mysql> select e.ename,(select d.dname from dept d where e.deptno = d.deptno) dname from emp e;//👇👇👇③

十一、union 合并查询结果集

1、格式：完整 select 语句① union 完整 select 语句②；

如：查岗位为manager和salesman的员工
①mysql> select ename,job from emp where job = ‘MANAGER’ or job = ‘SALESMAN’;
②mysql> select ename,job from emp where job in(‘MANAGER’,’SALESMAN’);
③mysql> select ename,job from emp where job = ‘MANAGER’ union select ename,job from emp where job = ‘SALESMAN’;

2、其中①和②为之前的语句，③为 union 语句，看着好像有点麻烦、多此一举，但对于计算机是效率更高的，尤其是当表的内容多的时候

若表a–>10条数据，表b–>10条数据，表c–>10条数据
在不用 union 时，计算次数为 a 连接 b 连接 c –>101010=1000次
在用 union 语句时，计算次数为 a 连接 b，a 连接 c –>1010+1010=200次

3、注意
mysql> select ename from emp where job = ‘MANAGER’ union select ename,job from emp where job = ‘SALESMAN’;//错误，两语句查询的列数要相同
mysql> select ename,job from emp where job = ‘MANAGER’ union select ename,sal from emp where job = ‘SALESMAN’;//错误，两语句查询的列名要相同

十二、limit

1、作用：将查询结果的一部分拿出来，通常用在分页查询中

2、格式：①limit x,y;// x 和 y 为数字，x 为起始下标，y 为查询长度，下标从0开始
②limit x;//取前x位
