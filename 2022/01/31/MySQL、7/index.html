六、insert 插入多条记录

1、格式：
insert into 表名(字段1,字段2,字段3…) values(第一条记录)，(第二条记录),…；

mysql> insert into t_user(id,name,birth,create_time) values(1,’zhangsan’,’1980-10-11’,now()),(2,’lisi’,’1981-10-11’,now()),(3,’wangwu’,’1982-10-11’,now());

2、表的复制

 格式：create table 表名 as select * from 被复制的表；

 创建一个内容与被复制表完全相同的新表

mysql> create table emp2 as select * from emp;

3、删除表中的数据

仅数据删除，表本身不删除

 ①delete from 表名；//DML语句

表中数据被删除，但数据在硬盘上占据的空间还在，可回滚，但删除速度慢，效率低

mysql> delete from dept_back;


  ②truncate table 表名；//DDL语句

表中数据被删除，一次性截断，物理删除，速度快，但不支持回滚

mysql> truncate table dept_back;

七、约束

1、在创建表时给字段加上约束，保证表中数据的完整性和有效性

2、分类
非空约束：not null
唯一性约束：unique
主键约束：primary key（PK）
外键约束：foreign key（FK）
检出约束：cheak

3、非空约束：not null

非空约束 not null 约束的字段其本身不能为 null

mysql> create table t_vip(id int,name varchar(255) not null);
Query OK, 0 rows affected (0.01 sec)
mysql> insert into t_vip(id,name) values(3);//发生错误， name 不能为空
ERROR 1136 (21S01): Column count doesn't match value count at row 1

形似 ***.sql 的文件称为 sql 脚本文件，可编写大量sql语句，用语句 source 文件路径；可快速执行文件内的语句

文本文档较小时，可打开复制粘贴，若文本文档很大，打开容易出问题，这时就要用到上面的语句


4、唯一性约束：unique

类似的，unique 要求内容不能重复（null 不算重复）

mysql> create table t_vip(id int,name varchar(255) unique,email varchar(255));

错误👇👇👇：‘zhangsan’已存在

联合：unique字段1,字段2,…)
各字段联合起来看作一个整体，只要整体符合条件，则其单个字段的内容就可以不被约束（如要求 not null 时可出现 null，要求 unique 时可出现两个‘zhangsan’）

如：现建表 t_ vip
mysql> create table t_vip(id int,name varchar(255),email varchar(255),unique(name,email));//这里联合 name 和 email

第一、二条记录成功录入，因为即使‘zhangsan’一样，但 ‘zhangsan-zhangsan@123.com‘ 和 ‘zhangsan-zhangsan@xina.com‘ 不一样
但第三条记录录入失败，因为连联合后的 ‘zhangsan-zhangsan@xina.com‘ 都一样

5、主键约束：primary key（PK）

将 not null 和 unique 一起用，即成为主键约束 primary key
mysql> create table t_vip(id int,name varchar(255) not null unique);


主键值：每一行记录的唯一标识（类似于身份证），主键值各不相同，即使主键值以外的其他所有内容都相同，也依然是两条完全不同的记录

任何一张表都应该要有主键，否则表无效

mysql> create table t_vip(id int primary key,name varchar(255));//列级约束
mysql> create table t_vip(id int,name varchar(255),primary key(id));//表级约束，二者效果相同

也可以多字段联合起来作复合主键（一个字段做主键叫单一主键）

mysql> create table t_vip(id int,name varchar(255)，email varchar(255),primary key(id,name));

mysql> create table t_vip(id int primary key,name varchar(255) primary key);//报错，主键只能有一个！！！
ERROR 1068 (42000): Multiple primary key defined

主键一般用在固定长度字段上，如 int，bigint，char 等

auto_increment 自增，在主键后加上，自动从1开始增加

mysql> create table t_vip(id int primary key auto_increment,name varchar(255));

6、外键约束：foreign key（FK）

foreign key(字段1) reference 表2(字段2)

将本表中字段1的输入限制在表2的字段2中。如表2的字段2的内容有1、2，那么本表的字段1只能输入1或者2，否则报错

其中本表称为子表，被引用的表称为父表

创建表、插入数据-->先父后子
删除表、删除数据-->先子后父

如：
mysql> create table t_class(classno int primary key,classname varchar(255));
mysql> create table t_student(no int primary key auto_increment,name varchar(255),cno int,foreign key(cno) references t_class(classno));

*外键字段引用其他表的某个字段的时候，被引用的字段不一定是主键，但至少具有unique约束。
*外键值可以为NULL。
